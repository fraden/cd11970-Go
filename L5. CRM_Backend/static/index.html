The CRM backend supports the following requests:

  • GET "/customers" to get all customers
  • GET "/customers/{id}" to get a specific customer, defined by id
  • PUT "/customers/{id}" to update a specific customer, defined by id
  • DELETE "/customers/{id}" to delete a specific customer, defined by id
  • POST "/customers" to create a new customer. You need to provide the customer data as json body.
A sample json body for the POST "/customers" request can look like this: 
        {
            "name": "Max Mustermann",
            "role": "solution architect",
            "email": "max.mustermann@coolmail.com",
            "phone": "0171/22446688",
            "contacted": false
        }
